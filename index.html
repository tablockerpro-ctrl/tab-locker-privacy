4. Permissions Explained

The Extension requires the following Chrome permissions:

  • storage: Save encrypted passwords locally on your device
  • activeTab: Read current tab URL when you click the extension icon
  • scripting: Inject the lock overlay on tabs
  • tabs: Detect when you switch tabs to trigger auto-lock
  • <all_urls>: Work on all websites (only checks if URL is locked)

What we DON'T do with these permissions:

  • We don't read page content
  • We don't track which websites you visit
  • We don't monitor your browsing behavior
  • We don't access tabs you haven't locked

8. Data Security

Your passwords are protected using:

  • AES-GCM 256-bit encryption
  • PBKDF2 key derivation (100,000 iterations)
  • SHA-256 cryptographic hashing
  • Random initialization vectors (IV) per encryption

All encryption happens locally using your browser's Web Crypto API.
